# Product Positioning Context

## Objetivo

Transformar as funcionalidades reais do sistema em uma narrativa comercial clara, objetiva e persuasiva.

## Diretrizes de copy

A landing page deve conter:

- Hero section com proposta de valor.
- Subtítulo explicando o sistema em linguagem simples.
- Lista de funcionalidades principais.
- Benefícios por perfil de usuário, se aplicável.
- Problemas que o sistema resolve.
- Diferenciais competitivos baseados no produto real.
- Como funciona.
- Seções de confiança.
- Perguntas frequentes.
- Chamadas para ação.
- Rodapé institucional.

## Técnicas permitidas

Use técnicas de copywriting como:

- Clareza antes de persuasão.
- Benefício antes de recurso.
- Mensagem orientada a dor, solução e resultado.
- Objeções e respostas.
- Prova por demonstração funcional.
- Frases diretas.
- CTA específico.
- Escaneabilidade.
- Hierarquia visual.

## Restrições

Não use:

- Promessas exageradas.
- Dados inventados.
- Depoimentos falsos.
- Números não comprovados.
- Garantias inexistentes.
- Claims absolutos como “o melhor”, “o único”, “100% garantido”, salvo se houver prova no projeto.
