# Documentation Standards

## Objetivo

Criar um manual de uso completo, claro e organizado para usuários finais.

## Estrutura obrigatória do manual

O manual deve conter:

1. Apresentação do sistema.
2. Público-alvo.
3. Requisitos básicos de acesso.
4. Login e autenticação, se existirem.
5. Visão geral da interface.
6. Explicação de cada módulo.
7. Passo a passo das funcionalidades.
8. Perfis de usuário e permissões, se existirem.
9. Fluxos principais.
10. Perguntas frequentes.
11. Resolução de problemas comuns.
12. Glossário.
13. Informações de suporte, se existirem no projeto.

## Linguagem

Use linguagem clara, instrutiva e objetiva.

Evite linguagem excessivamente técnica para usuário final.

## Formatos finais

Gerar:

- `public/manual/manual.html`
- `public/manual/manual.pdf`

Se o projeto tiver outra estrutura de publicação, adaptar preservando a intenção.
