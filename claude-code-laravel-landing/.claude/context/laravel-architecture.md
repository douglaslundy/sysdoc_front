# Laravel Architecture Context

## Áreas a analisar

Verifique obrigatoriamente:

- `routes/web.php`
- `routes/api.php`
- `app/Http/Controllers`
- `app/Models`
- `app/Policies`
- `app/Services`
- `app/Actions`
- `app/Jobs`
- `app/Notifications`
- `database/migrations`
- `database/seeders`
- `resources/views`
- `resources/js`
- `resources/css`
- `resources/components`
- `tests`
- `config`
- `README.md`
- Outros arquivos de documentação existentes.

## Pontos técnicos obrigatórios

Identifique:

- Framework e versão aproximada.
- Stack frontend usada.
- Sistema de autenticação.
- Perfis ou papéis de usuário.
- Rotas públicas.
- Rotas protegidas.
- Dashboards.
- CRUDs.
- Integrações externas.
- Relatórios.
- Uploads.
- Notificações.
- Pagamentos, se existirem.
- APIs, se existirem.
- Filas/jobs, se existirem.
- Permissões e policies.
- Entidades principais do domínio.
